Git Command Basics

1. git init

This command initializes a new Git repository in your current directory. It creates a hidden .git folder which stores all the configuration and version history.

Usage: git init

2. git add

This command adds files to the staging area, meaning you're telling Git to include updates to these files in the next commit.

Usage: git add filename or git add . (to add all changes)

3. git commit

This command saves your changes to the local repository. It takes everything from the staging area and stores it as a snapshot in your Git history.

Usage: git commit -m "Your commit message"
